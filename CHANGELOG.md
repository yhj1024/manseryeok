# Changelog

본 프로젝트는 [Semantic Versioning](https://semver.org/lang/ko/)을 따릅니다.

## [2.0.0]

정확도 전면 재정비 및 명리 기능 확장. 일부 출력이 1.x와 달라지는 **breaking change**가
포함됩니다(대부분 1.x의 계산 버그를 바로잡은 결과입니다). 아래 마이그레이션을 참고하세요.

### ⚠️ Breaking Changes (정확도 수정)

- **연주(年柱) 입춘 보정**: 1.x는 연주를 양력 연도로만 계산해, 입춘(2/4경) 이전 출생자의
  연주가 1년 틀렸습니다. 이제 입춘 절입 순간을 경계로 올바르게 계산합니다.
  - 예: `2000-01-01` → 1.x `경진년`(오류) → 2.0 `기묘년`(정답, 입춘 전).
  - 예: `2024-01-14` → 1.x `갑진년`(오류) → 2.0 `계묘년`(정답).
- **절기(節氣) 정밀화**: 거친 선형 근사식을 제거하고, 1800~2300년은 임베드 정밀 절입표
  (6tail 천문 계산값을 개발 시점에 추출, KASI와 분 단위 일치)로 연주·월주를 판정합니다.
  범위 밖은 Meeus 근사로 폴백합니다(런타임 의존성 추가 없음).
  - 예: 2024 입춘 = `2024-02-04 17:27 KST` 기준. 17:26 출생은 계묘년, 17:28 출생은 갑진년.
- **`solarToLunar` 정정**: 1.x에서 깨져 있던(테스트가 `skip`되어 있던) 양력→음력 변환과
  윤달 처리를 수정했습니다. 이제 `lunarToSolar`의 정확한 역함수입니다.
- **타임존 안정성**: 날짜 계산을 UTC 기반으로 변경해, 실행 환경(서버) 타임존에 따라
  결과가 달라지던 잠재 버그를 제거했습니다.
- **입력 검증 강화**: 실재하지 않는 양력 날짜(`2024-02-31`, 윤년 아닌 `2-29` 등),
  `NaN`/정수 아님/지원 범위(양력 1800~2300·음력 1800~2100) 밖 연도, 범위를 벗어난 월/일/시/분,
  존재하지 않는 천간·지지 조합(예: 갑축)은 이제 조용히 `NaN`을 내지 않고 `RangeError`를
  던집니다. `solarToLunar`도 동일하게 검증합니다. (`isValidSolarDate` 공개)
- **빌드 산출물**: ESM(extensionless) → **CommonJS**로 변경(`require`/`import` 모두 지원,
  번들러 호환). `package.json`에 `exports` 필드를 추가하고 `module` 필드를 제거했습니다.
- **파일 구조**: 모든 소스 파일/모듈명을 kebab-case로 통일했습니다(공개 API 영향 없음).

### ✨ Added

- **진태양시(眞太陽時) 보정 — 옵션, 기본 OFF**: `BirthInfo.trueSolarTime`으로
  경도 보정 + 균시차(EoT) + 과거 표준시/서머타임(1948–88, 1954–61 UTC+8:30)을 적용.
  미지정 시 기존처럼 KST 표준시 그대로 계산합니다(하위호환).
- **야자시/조자시(`dayBoundary`)**: 23:00~23:59 자시의 일주·시주 천간 기준을 세 관법으로
  선택 — `'midnight'`(기본, 둘 다 당일) / `'jasi'`(둘 다 다음날) / `'splitJasi'`(일주는 당일,
  시주 천간만 다음날, 6tail 기본 방식).
- **명리 부가 정보**: 결과 객체에 `tenGods`(십신), `voidBranches`(공망) 포함.
  `gender` 지정 시 `luckPillars`(대운) 포함.
  (해석이 학파마다 갈리는 신살(神煞)·12운성은 결과 신뢰성을 위해 제공하지 않습니다.)
- **대운 세부 시작 시점**: `luckPillars`에 `startYears`·`startMonths`·`startDays`를 추가해
  연 단위로 반올림되는 `startAge` 외에 월·일 잔여를 보존합니다(3일=1년, 1일=4개월 환산).
  일=30/월=12 경계는 상위 단위로 올림(carry) 정규화합니다.
- **사주 지원 범위 확장**: 양력 1800~2300년, 음력 1800~2100년. 두 입력 모두 정밀 절입표
  범위(1800~) 안에서만 계산해 전 구간이 분 단위로 정확합니다.
- **음력 테이블 KASI 정본화 + 변환 범위 확장(1391~2100)**: 음력 데이터를 한국천문연구원(KASI)
  음양력 API 기준으로 재생성했습니다(1391~2049 KASI, 2050~2100 천문 계산).
  - **한·중 차이 교정**: 삭(신월)이 자정 근처라 한국(KST)·중국(CST) 음력이 갈리는 해
    (1900~2049 약 43개)에서, 기존 중국 기준 값을 **한국 정본으로 교정**했습니다.
    예: 1997 설날 = 2/8(한국)·2/7(중국). 일주(60갑자)는 KASI 일진과 1391~2049 전 구간 일치 검증.
  - **음양력 변환**: `lunarToSolar`/`solarToLunar`는 조선·고려말(1391~)까지 지원합니다
    (사주 계산 입력은 절기 정밀도를 위해 1800년부터).
- **공개 API 추가**: `getSolarTerm`, `getSolarTermsOfYear`, `apparentSolarLongitude`,
  `equationOfTimeMinutes`, `getTenGod`, `getTenGodChart`, `getVoidBranches`,
  `getLuckPillars`,
  `getEarthlyBranchYinYang` 등.

### 🧪 검증

- 널리 쓰이는 `6tail/lunar-javascript`와 60갑자 핵심 로직을 교차검증(자시·절입 경계 제외 시
  ≥98% 일치, 불일치는 모두 KST↔CST 절입 경계로 설명됨)하는 테스트를 추가했습니다.
- 1.x에서 틀린 값을 고정하던 테스트를 검증된 정답으로 교체하고, `skip`되어 있던
  음력 변환/윤달 테스트를 복구했습니다.
- **음력 테이블 월 대소(大小) 정정**: 1933·1996·2060년의 월 길이 데이터를 삭(신월) 절대
  시각 기준으로 바로잡았습니다(예: 음력 1996-06-01 → 양력 7/16). 정정 후 1900~2100년 전
  구간이 천문학적 삭 시각과 일치합니다.
